Feedback Page

Technical Description

The Feedback Page allows users to share their experiences and suggestions. It is built using HTML and CSS, ensuring a clean and structured layout. The page follows a structured div container model with a dedicated header, main, and footer section for logical content flow.

The feedback form includes text inputs, radio buttons, dropdowns, and a textarea. CSS is used for consistent styling, while JavaScript provides validation to ensure required fields are filled. The form prevents incorrect submissions by checking email format and required inputs, improving user interaction and data accuracy.

Accessibility

The Feedback Page is designed with accessibility in mind. Labels are linked to form fields for screen reader compatibility, and alternative text (alt) is provided for images. The page maintains proper contrast ratios between text and background for readability. All form elements support keyboard navigation, allowing users to tab through inputs and buttons efficiently.

Font sizes are defined using relative units (rem and em) to support text resizing. Error messages provide clear guidance for users with disabilities.

Team Page

Technical Description

The Team Page introduces the project team members, showcasing their roles and contributions. The page structure follows a container-based model, ensuring a clean and organized layout using HTML and CSS. The main content is wrapped in a div with class "team-rectangle" for styling and structure.

The page features a grid-based layout with div elements representing individual team members. Each member is displayed with an image, name, role, and a list of implemented pages. Hover effects reveal additional details about each member using CSS transitions. Flexbox and grid are used to ensure a responsive design, making the layout adaptable to various screen sizes.

Accessibility

The Team Page is designed with accessibility in mind. Images have alternative text (alt) to support screen readers, and proper color contrast is maintained for readability. The hover-based information display is complemented by structured HTML, ensuring that users relying on assistive technologies can access the content effectively.

Font sizes are set using relative units (rem and em) to support text resizing. The layout remains user-friendly across different devices, ensuring ease of interaction.

Content Page

Technical Description

The Content Page 3 focuses on "The Role of Individuals and Communities in Fighting Hunger." It follows a structured layout with sections for different aspects of combating hunger. The content is wrapped in a div with class "content" for styling and readability.

The page is designed with a hierarchical structure using h1 and h2 tags to improve readability and SEO. Internal navigation links help users jump to different sections easily. CSS employs flexbox and grid to ensure a responsive design, allowing for seamless adjustments across different screen sizes.

Accessibility

The page is optimized for accessibility by including alt attributes for all images, ensuring screen reader compatibility. Text content is structured with proper heading levels for better navigation.

The navigation menu and internal links provide a user-friendly browsing experience, allowing visitors to access relevant information quickly. Font sizes are set using relative units (rem and em) to support scalability.

Challenges and Lessons Learned

Challenges

During the development process, one of the main challenges encountered was maintaining consistency across all web pages. Since different team members worked on various sections, ensuring uniformity in design, navigation, and styling was critical. A global CSS file was shared among all team members to maintain consistency. Additionally, ensuring the accessibility of interactive elements, such as buttons and icons, presented challenges. For example, some custom buttons were not fully accessible to screen readers, requiring us to make adjustments.

Another challenge was fine-tuning the navigation, particularly the dropdown menus and button controls. We also needed to ensure that dynamic JavaScript interactions, like expandable reviews, functioned well without negatively affecting usability or accessibility when the content changed.

Lessons Learned

From these challenges, we learned the importance of collaboration and organized file management, especially when working with a team. Clear communication and frequent code sharing were crucial to keep the project on track. One major lesson was the importance of testing early and often to identify issues across various devices and browsers before they became blockers. While embracing modern design trends, we had to ensure that accessibility and compatibility were not compromised, especially for users with disabilities or those using different devices.

We also reinforced the significance of writing clean and structured code. Troubleshooting validation errors and maintaining compliance with HTML and CSS standards enhanced our attention to detail and improved the quality of the website.

Compliance with Janet Regulations

Compliance Overview

The pages we created comply with Janet’s regulations governing web page publication. This includes adherence to accessibility standards, HTML/CSS validation, and best practices for user experience.

Accessibility Compliance

We ensured that the pages are accessible to users with disabilities by including alt attributes for all images and optimizing interactive elements for keyboard navigation. Proper heading hierarchy was used to improve navigation for screen readers, and color contrast was considered for readability.

HTML/CSS Standards

All pages were validated for proper HTML and CSS structure, ensuring that there are no errors or issues that would affect functionality or display. Compliance with these standards helps improve cross-browser compatibility and overall page performance.

User Experience

The site was designed with a focus on user experience, ensuring that the navigation is intuitive, and all content is easily accessible. The use of relative font sizes (rem and em) helps users scale the page for better readability on various devices.
