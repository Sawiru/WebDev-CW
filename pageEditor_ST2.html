Home page

Technical Description

The Home Page serves as the main entry point for the website and sets the visual and structural foundation. The page is designed using HTML and CSS, ensuring a clean, structured, and responsive layout. The navigation bar is implemented using an unordered list and styled with CSS to provide hover effects and visual feedback on active links. The page layout follows a structured div container model with a dedicated header, main, and footer section, ensuring a logical content flow.

Interactive elements such as buttons and hyperlinks are styled with CSS transitions to improve user experience. The homepage also includes an introductory section with an eye-catching heading, a brief description of the project's mission, and interactive tiles linking to different content sections. These tiles employ CSS animations, such as hover effects and box shadows, to enhance usability and engagement.

Accessibility

The Home Page is designed with accessibility in mind, following guidelines given. Alternative text (alt) is provided for images, ensuring that screen readers can interpret visual content. Proper contrast ratios between text and background are maintained for readability. The navigation bar is fully keyboard-accessible, allowing users to tab through links effectively.

Font sizes are defined in relative units (rem and em) to support text resizing for users with visual impairments. A "Go to Top" button is included, making navigation easier for users browsing long pages.

Table

Technical Description

The Table Page features a structured and interactive table presenting relevant data in an easy-to-read format. The table is built using the <table> element, incorporating <thead>, <tbody>, and <tfoot> to ensure a clear and structured hierarchy. Column headers (<th>) are styled with CSS to make them visually distinct, while alternating row colors (nth-child CSS selector) improve readability.

Icons are used within the table cells to visually represent key data points. The hover effects on rows provide a subtle interaction cue, improving user engagement. A scrollable table wrapper is implemented to handle large datasets without breaking the layout.

Accessibility

Each table cell has a meaningful label, and tooltip descriptions (title attribute) are added to provide additional information when hovered over.

For better contrast, table rows use alternating colors, and high-contrast borders help distinguish content effectively. Additionally, zoom support is tested to ensure users with visual impairments can easily enlarge content without breaking the page layout.

Content Page

Technical Description

The Content Page is structured to present detailed information in an engaging and readable manner. It utilizes a multi-section layout, incorporating HTML elements like <section>, for better content organization. Internal linking is implemented through anchor (<a>) elements with IDs, allowing users to navigate to specific sections without reloading the page.

CSS is used to create a well-structured layout with consistent typography and spacing. The page also features an interactive content navigation menu that remains fixed (position: fixed) to enhance usability. Interactive elements such as images, icons, and buttons are styled with CSS hover effects, transitions, and shadow properties to provide a modern look and feel.

Accessibility

To enhance accessibility, all images and icons have appropriate alt text. The page uses semantic HTML elements, making it easier for screen readers to interpret the content structure. For navigation, a skip link is implemented at the top of the page, allowing users to jump directly to the main content. Internal links are accompanied by ARIA labels where necessary to improve usability for assistive technologies.

Headings are structured in a logical hierarchy (h1 followed by h2, h3, etc.), improving readability. The "Go to Top" button is included to aid navigation. Additionally, content sections have sufficient line height and padding, ensuring a comfortable reading experience for all users.

Challenges and Lessons Learned

Challenges

During the development of the website, several challenges arose, requiring problem-solving and adaptation. One of the primary difficulties was ensuring consistency across all web pages, as different team members were responsible for different sections. Maintaining a uniform design, navigation, and styling required effective communication and adherence to a global CSS file. Additionally, implementing responsive design posed a challenge, particularly in ensuring that elements adapted correctly across different screen sizes without breaking the layout. Another hurdle was validating all pages to comply with HTML and CSS standards, as minor syntax errors or incorrect attribute usage led to unexpected display issues.

Lessons Learned

Through these challenges, we learned the importance of collaboration and version control, as sharing code and making consistent updates required clear communication and structured file management. We also gained valuable experience in web accessibility, keyboard navigation, and proper contrast ratios contribute to an inclusive user experience. Troubleshooting validation errors improved our attention to detail, reinforcing the necessity of writing clean and structured code. Overall, the project strengthened our skills in HTML, CSS, and responsive design, while also highlighting the significance of teamwork in web development.

Compliance

The pages I have created are compliant with Janet regulations governing web page publication. This includes ensuring that all pages are validated for HTML and CSS standards, and that accessibility guidelines are followed. The pages are designed to be inclusive, with proper alt attributes for images, keyboard navigation, and optimized color contrast. The content is original and relevant to the theme of Zero Hunger, and all external resources are properly referenced.
