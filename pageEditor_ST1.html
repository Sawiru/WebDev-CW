Splash Page

Technical Description

This page serves as an introductory splash screen, providing a brief introduction before automatically redirecting the user to the main homepage after 4 seconds. It sets the visual tone for the website, providing users with an engaging first impression. The centered layout provides a balanced and visually appealing view, making the content easy to read. height: 100vh; Ensures the splash screen takes up the full height of the viewport, creating a full-screen effect for the introduction and the warm orange color used for headings and the gradient background color evoke feelings of warmth which ties in perfectly with the theme of Zero Hunger. Class "content-wrapper" container wraps all content, applying max-width, padding, and a background gradient, ensuring the content doesn't stretch too far across wide screens while maintaining an aesthetically pleasing design. Class "loading-container" Holds the spinner animation, giving users a visual cue that the page is loading and they should wait for the redirection. The Montserrat font keeps the design consistent, readable, and modern.The loading animation helps users understand that the page is being processed, improving user experience. By using an external CSS file, updates and maintenance are easier. The spinner is created using CSS properties like border-top and border-radius to form a circular spinning effect and @keyframes spin defines the continuous rotation of the spinner to give it a smooth animation.

Accessibility

The splash page uses several accessibility features to ensure an inclusive experience. It employs semantic HTML, such as h1, h2, and p tags, for proper content structure and readability. The background and text offer sufficient contrast for users with visual impairments, while alt text for images ensures screen readers can describe the content.

Volunteer Page

Technical Description

This webpage is designed to apply for the zero hunger volunteering Programs. A container holds the main content, class="container" wraps the main content for organization. class="scrollable-section" enables horizontal scrolling for volunteer programs, with 4 cards featuring images, descriptions, and buttons for interaction. A dropdown allows choosing an option how to view the cards. class="reviews" holds user reviews with a "See More" option for expanded content using JavaScript. The feedback form includes emoji-based ratings and text input for user engagement. CSS employs flexbox and grid for layout, display: flex; used in .scrollable-section, .reviews, and .feedback-emoji to create a flexible, responsive layout.

Accessibility

The page enhances accessibility by allowing users to easily scroll through volunteering events using a horizontally scrollable section, making it convenient to browse multiple options without excessive clicking. The dropdown menu enables sorting options like "Most Popular" and "Newest," improving user experience by helping them find relevant opportunities quickly. Each event card includes a image, title, description, and a button for further details, ensuring readability and ease of interaction. The feedback form is designed with accessibility in mind, offering both text input and emoji-based ratings for an engaging user experience. Buttons are large and well-spaced for easy selection, and hover effects provide visual feedback, making the interface more interactive and user-friendly across different devices.

Content Page

Technical Description

The Content Page is designed for clear and engaging information presentation, utilizing a structured multi-section layout. It uses elements such as (section, aside, and figure) to improve content organization. Internal navigation is facilitated through anchor (a) elements with IDs, allowing seamless movement between sections without requiring a page reload. CSS ensures a visually appealing and consistent layout by maintaining uniform typography and spacing. A fixed-position navigation menu enhances user experience by staying accessible while scrolling. Interactive elements, icons, and buttons, feature hover effects, transitions, and shadow properties for a modern and dynamic appearance.

Accessibility

To improve accessibility, all images and icons include descriptive alt text. The use of semantic HTML elements enhances readability for screen readers by providing a clear content structure. Navigation links are positioned at the top, enabling users to quickly access the main content. Headings follow a logical hierarchy (h1, followed by h2, h3, etc.), ensuring better readability. A "Go to Top" button is included for easier page navigation. Additionally, content sections are designed with adequate line height and padding to enhance readability and user comfort.

Challenges and Lessons Learned

During the process, one of the main challenges encountered was maintaining consistency across all pages, as various team members worked on different sections. To ensure the design stays the same, navigation, and styling, we shared a global CSS file which was essential. Additionally while I made sure to use semantic HTML and added appropriate attributes, there were some interactive elements that didn’t behave as expected. For instance, custom buttons with icons were not fully accessible to screen readers. I also had to fine-tune the navigation parts. This was especially important for the dropdown and button controls. Additionally, integrating the dynamic JavaScript interactions like the expandable reviews required ensuring that the page remained usable and accessible when content changed. Lessons learned from these challenges was the importance of thorough testing across devices and browsers and to keep the work organized especially because you're sharing with your team mates aswell. Additionally, while implementing modern design trends, one must not overlook the core principles of accessibility and compatibility, especially when designing for a wide range of users. Testing early and often was essential in identifying issues before they became blockers.
